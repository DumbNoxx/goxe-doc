Goxe transforms noisy log streams into actionable insights. 
This guide will help you configure your first pipeline to collapse thousands of repetitive entries into smart summaries,
cutting storage costs immediately.

## How to get started?
### Install Goxe
First you need to install Goxe; you can install it from different sources.

<Tabs>
  <TabItem label="Go" icon='seti:go'>
    <p>If you have Go installed. Recommended Option</p>
    <Steps>
      1. Run the following command to download and build the project:
          ```bash title="Terminal" 
          go install github.com/DumbNoxx/Goxe/cmd/goxe@latest 
          ```
          <p>Goxe is a standalone binary with no external dependencies just download and run.</p>
      2. Verify the installation
          ```bash title="Terminal"
          goxe -v
          # v1.1.2
          ```
      3. Run Goxe
          ```bash title="Terminal"
          goxe
          # Server listening on port: 1729
          ```
    
    </Steps>
   </TabItem>
  <TabItem label="Linux" icon="linux">
    <p>In Linux you have different ways to install it.</p>
    <Tabs>
      <TabItem label="Ubuntu/Debian">
        <Steps>
      1. Download the file corresponding to your PC architecture.
          <ButtonDownload extension="deb" os="linux" architecture="amd" />
          <ButtonDownload extension="deb" os="linux" architecture="arm" />

      2. The next step would be to double-click or use:

          ```bash title="Ubuntu Shell"
          sudo apt install /path/to/Goxe_1.1.2_linux_amd64.deb # Or arm64
          ```


      <TabItem label='Arch Linux/Manjaro'>
      <p>In the case of Arch Linux you have two options temporarily</p>
      <p>As with Ubuntu, Goxe isn't in the official repositories, but it is in the AUR.</p>
        <Steps>
      1. Choose the helper you use.
          <Tabs>
          <TabItem label='yay'>
            ```bash title="Arch Shell"
            yay -S goxe
            ```
          </TabItem>
           <TabItem label='paru'>
            ```bash title="Arch Shell"
            paru -S goxe
            ```
          </TabItem>
          <TabItem label='AUR (Pre-built)'>
          <p>Goxe also has its pre-compiled binary in the AUR, so you can install it with paru or yay.</p>
            ```bash title="Arch Shell"
            yay -S goxe-bin
            ```
            <Aside>
              If you use this version, remember to replace <code>goxe</code> with <code>goxe-bin</code> in the following steps.
              Also, keep in mind that this version is already pre-compiled, so it will skip the build process entirely.
            </Aside>
          </TabItem>
          </Tabs>
      2. Confirm the installation.
          <p>
            Your AUR helper will automatically download the source code and compile it. 
            Since <strong>Goxe</strong> has no external dependencies, the process is 
            clean and fast. Simply press <code>Enter</code> when prompted for 
            installation confirmations.
          </p>
      3. Verify the installation:
          ```bash title="Arch Shell"
          goxe -v
          # v1.1.2 
          ```
          The latest version should be displayed.
      4. Run goxe
          ```bash title="Arch Shell"
          goxe
          # Server listening on port: 1729
          ```
        </Steps>
      </TabItem>
    <TabItem label='Other Linux'>
  <p>The pre-compiled binary is available for direct download in <code>.tar.gz</code> format.</p>

  <ButtonDownload extension="tar.gz" os="linux" architecture="amd" />
  <ButtonDownload extension="tar.gz" os="linux" architecture="arm" />

  <Steps>
    1. Create the destination folder

        ```bash title="Terminal"
        mkdir -p /destination/goxe
        ```
        <Aside>
          This directory will store the extracted binary. The executable will be exposed globally using a symbolic link.
        </Aside>

    2. Extract the archive

        ```bash title="Terminal"
        tar -xzvf Goxe_1.1.2_linux_amd64.tar.gz -C /destination/goxe # Or arm64
        ```
        <p>
          Use the same directory you created in the previous step.
        </p>
    3. Make the binary executable and create a symbolic link

        ```bash title="Terminal"
        chmod +x /destination/goxe/Goxe
        sudo ln -s ~/destination/goxe/Goxe /usr/local/bin/goxe
        ```
        <Aside type='caution'>
          Creating the symbolic link requires <code>sudo</code> permissions.
        </Aside>

    4. Run Goxe

        ```bash title="Terminal"
        goxe
        # Server listening on port: 1729
        ```
        <p>
          The symbolic link allows you to run <code>goxe</code> globally while keeping the binary in its own directory.
        </p>
  </Steps>
</TabItem>

<TabItem label="Windows" icon="seti:windows">
  <p>In Windows, you can download the pre-compiled binary in <code>.zip</code> format.</p>

  <ButtonDownload extension="zip" os="windows" architecture="amd" />
  <ButtonDownload extension="zip" os="windows" architecture="arm" />

  <Steps>
    1. Extract the downloaded archive to your desired location.

    2. (Optional) Add Goxe to your system PATH for global access:
          - Open the Start Menu and search for "Environment Variables".
          - Select "Edit the system environment variables".
          - Click on "Environment Variables".
          - Under "System variables", find and select the "Path" variable, then click "Edit".
          - Click "New" and add the path to the directory where you extracted Goxe.
          - Click "OK" to close all dialog boxes.
  </Steps>

        3. Double-click the <code>goxe.exe</code> file.

            ```text
            Server listening on port: 1729
            ```

<TabItem label="MacOS" icon="apple">
  <p>The pre-compiled binary is available for direct download in <code>.tar.gz</code> format.</p>

  <ButtonDownload extension="tar.gz" os="darwin" architecture="amd" />
  <ButtonDownload extension="tar.gz" os="darwin" architecture="arm" />

  <Steps>
    1. Create a directory for the binary

        ```bash title="Terminal"
        mkdir -p ~/goxe
        ```

        <Aside>
          This directory will store the Goxe binary. A symbolic link will be created for global access.
        </Aside>

    2. Extract the archive

        ```bash title="Terminal"
        tar -xzvf Goxe_1.1.2_macos_amd64.tar.gz -C ~/goxe
        ```

    3. Make the binary executable and expose it globally

        ```bash title="Terminal"
        chmod +x ~/goxe/Goxe
        sudo ln -s ~/goxe/Goxe /usr/local/bin/goxe
        ```

        <Aside type="caution">
          Creating the symbolic link requires <code>sudo</code> permissions.
        </Aside>

    4. If macOS blocks the binary, remove the quarantine flag

        ```bash title="Terminal"
        xattr -d com.apple.quarantine ~/goxe/Goxe
        ```

        <Aside>
          This is required for unsigned binaries downloaded from the internet.
        </Aside>

    5. Run Goxe

        ```bash title="Terminal"
        goxe
        # Server listening on port: 1729
        ```
  </Steps>
